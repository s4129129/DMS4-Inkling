# UI Localization

Inkling currently supports English and Vietnamese. User-facing static copy should
flow through `src/i18n/useVietnameseDomTranslation.js`.

## Rules

- Add new Vietnamese strings to `EXACT_TRANSLATIONS` when the English source is
  stable.
- Add dynamic strings to `REGEX_TRANSLATIONS` when the UI includes changing
  values such as names, counts, or titles.
- For non-DOM text such as chart labels, canvas labels, or computed labels,
  import `translateUiText(value, language)` from `src/i18n`.
- Do not add small component-local translation maps. They drift and make missing
  copy hard to find.

## Audit

Run this before shipping localization-heavy changes:

```powershell
npm run i18n:audit
```

The audit lists obvious JSX text, accessible labels, placeholders, and status
messages that do not have a Vietnamese translation yet.
